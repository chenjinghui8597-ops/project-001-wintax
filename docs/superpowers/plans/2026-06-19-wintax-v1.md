# WINtax V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and package the complete first version of WINtax as a local Windows desktop application for digital export-tax filing document management.

**Architecture:** Use Electron Forge with the Vite TypeScript template. React owns the renderer UI, a narrow preload API exposes typed IPC calls, and the Electron main process owns SQLite and all filesystem operations. Store metadata in SQLite and attachment files in the user-selected local repository; implement each business rule as a small service with unit tests before wiring it to IPC and UI.

**Tech Stack:** Electron 42, Electron Forge 7, React 19, TypeScript 6, better-sqlite3 12, read-excel-file 9, archiver 8, Vitest 4, React Testing Library, Playwright, bcryptjs, Zod.

---

## Success Criteria

- All required features in `docs/superpowers/specs/2026-06-19-wintax-local-filing-design.md` are implemented.
- `npm.cmd run lint`, `npm.cmd run typecheck`, `npm.cmd test`, and `npm.cmd run make` pass.
- A clean Windows installation can initialize a repository, log in, import a valid plan, reject an invalid plan, manage attachments, complete filing, export ZIP files, back up, restore, and manage users.
- The installer is produced under `out/make/` and copied to `output/`.
- Source code, tests, README, and build instructions remain in the project.
